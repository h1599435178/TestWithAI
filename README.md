# Test with AI 开发手册 v0.0.1

## 1. 用来做什么
这个项目的初衷，是用来搭建一个内置OpenClaw的自动化测试助手。
在OpenClaw面世以来，笔者就一直在尝试使用OpenClaw来接入测试环境，但始终无法解决一个痛点，就是Token消耗爆炸。这个项目旨在解决这个问题进行开发。
目前规划了这些事情：
### 1.1 本地部署
该项目支持本地部署，用户可以拖拽包、上传网站来实现自动化。
### 1.2 Skill支持
支持安装、自定义Skills，并内置了多个自动化可用的Skills以支持项目运行。
### 1.3 RAG管理
支持读取需求文档、Mock截图保存等操作，以节省token消耗。
### 1.4 跨平台支持
目前规划了Web端、Android端的测试支持，未来会继续跟进iOS端的开发。
### 1.5 MCP支持
支持多MCP集成，打通平台生态。
### 1.6 模型支持
用户可以自定义模型Endpoint、BaseUrl、ModelID等参数，来选择自己想要的服务商以及模型，用最优惠的价格使用该产品。
### 1.7 Prompt定义
用户可以自定义Prompt，来满足不同测试场景的不同需求。
### 1.8 配套生态
目前规划支持腾讯Tapd平台，用来在测试后直接返回缺陷，提交到对应项目上。用户需选择对应的Tapd项目，才能正常使用。

## 2. 能进行哪些测试
### 2.1 功能测试
支持对功能模块的测试，包括但不限于接口测试、UI测试等。
### 2.2 性能测试
支持对系统性能的测试，包括但不限于响应时间、吞吐量等。
### 2.3 安全测试
支持对系统安全的测试，包括但不限于SQL注入、XSS攻击等。
